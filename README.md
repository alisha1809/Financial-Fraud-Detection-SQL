# Financial-Fraud-Detection-SQL

## Overview

This project analyzes financial transaction data using SQL to identify suspicious transaction patterns and support fraud detection.

The project demonstrates SQL querying, data analysis, and relational database concepts commonly used in banking and financial analytics.

---

## Tech Stack

- MySQL
- SQL
- Python
- Microsoft Excel

---

## Project Objectives

- Detect rapid-fire transactions
- Identify geographic anomalies
- Analyze unusual spending patterns
- Generate fraud-related business insights

---

## Dataset

The dataset contains 100+ financial transaction records.

Columns include:

- Transaction ID
- User ID
- Transaction Amount
- Transaction Time
- City
- Merchant
- Payment Method

---

## SQL Concepts Used

- SELECT
- WHERE
- GROUP BY
- HAVING
- ORDER BY
- Aggregate Functions
- INNER JOIN
- SELF JOIN
- Window Functions
- Subqueries

---

## Project Structure

Database.sql

Queries.sql

Business insights generated from the analysis.

## Sample Insights

- User 101 completed multiple transactions within one minute.
- Multiple users performed transactions from different cities within short time intervals.
- High-value transactions were identified for further fraud investigation.
